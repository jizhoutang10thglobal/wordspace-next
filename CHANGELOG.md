# Changelog

> **约定**（2026-07-12 起）：每次发版（打 tag）后在这里给该版本写一段**人话说明**（用户视角优先，开发侧改动一笔带过），
> 并把同一段同步到对应 GitHub Release 页的顶部（Release 页保留自动生成的 PR 列表给开发者看细节）。
> 正本是本文件；历史版本（v0.0.1–v0.6.6）为 2026-07-12 回溯补写。
> 版本号：tag 即版本真源（`git tag vX.Y.Z && git push origin vX.Y.Z` 触发签名发版），带新功能 bump 0.1，纯修复 bump patch。

## v0.10.4 — 2026-07-17

**PDF 可以默认用 Wordspace 打开了，外加三个日常修复：更新弹窗不再盖白网页、「重启安装」二修、标签不再误标「工作区外」。**

- **PDF 默认打开可以设成 Wordspace 了**：macOS「打开方式 → 全部更改」的候选列表里现在有 Wordspace（此前安装包没声明 .pdf 文档类型，列表里根本不出现）；设为默认后双击 PDF 直接在应用内查看器打开。
- **更新弹窗不再把网页盖白**：网页标签（比如 Gmail）开着时弹出「发现新版本」，内容区会整个变白——现在弹窗背后垫了页面快照，网页内容保持可见。
- **「重启安装」二修**：v0.10.1 修过一次但没修干净——点了之后窗口消失、app 却赖在 Dock 里，更新装不上。这次找到真根因（退出信号的监听挂错了对象、从来没生效过），把退出动作直接放到按钮上。老规矩：装上本版的这一次更新仍走旧更新器（可能还要手动退出一次），**从本版起的下一次更新**才是真正的一键重启。
- **标签不再误标「工作区外」**：先打开一个单独文件、再把它所在的文件夹添加进工作区，标签页里这个文件此前会一直带着 ↗「工作区外」标记（Wendi 报的），还会因为身份错乱出现同一文件两条标签。现在添加文件夹时自动收编；老版本已经留下的错误标签，升级后启动即自愈，无需手动处理。
- 开发侧：外部标签收编挂到全部「树内容到货」点（含简化模式逐层加载）；两条 e2e 新门（操作流收编 + 启动自愈）先红后绿 + 逐点变异复核；PDF 文档类型声明有打包层单测兜底。

## v0.10.3 — 2026-07-16

**超大文件夹能正常用了（简化模式），外加整个 app 支持中英文切换。**

- **超大文件夹进「简化模式」**：上一版（v0.10.2）让巨型文件夹不再卡死但打不开内容；这一版能**正常浏览**了——超过 15 万个项目的文件夹自动进入简化模式（根上带徽标），秒级打开、点开哪层加载哪层，照常打开和编辑里面的文档，重启后依旧秒开。实测整个用户目录（约 190 万个项目）从选择到可浏览约 2 秒。
- **简化模式下的取舍（有提示，不静默）**：侧栏筛选只搜浏览过的部分；快速打开（Cmd+P）暂不覆盖这类文件夹；外部改名的标签跟随降级为按路径匹配。
- **大文件夹的后台负担大幅下降**：磁盘变化只重读你浏览过的目录，高频写入的大文件夹不再触发全量重扫；@链接菜单不再钻进 node_modules / .app 包内部（更快也更准）。
- **中英文切换**：设置页新增「语言」——跟随系统（默认）/ 中文 / English，切换后整窗重载生效；菜单栏、对话框、提示等全量覆盖。英文文案为初版，术语仍在打磨中。
- 开发侧：文件树构建消除 O(M²)（15 万条目宽树 68s → 0.6s）；诊断面板补目录数与载荷指标；i18n 三道防漂移门接入 CI；两轮对抗审查（真实 191 万条目家目录重放全过）。

## v0.10.2 — 2026-07-16

**界面「零缝隙」沉浸模式（对标 Arc），外加超大文件夹不再把 app 冻死。**（补账：本段与 Release 页同步）

- **系统标题栏没了（macOS）**：窗口顶上那条「Wordspace Next」标题栏取消，红绿灯按钮融进侧栏顶部——窗口从第一个像素起就是内容；侧栏顶部区域可以拖动窗口。（Windows 暂保持原窗框）
- **收起侧栏 = 完全沉浸**：收起后左侧不再留任何窄条或按钮，内容（尤其网页标签）铺满整个窗口，红绿灯也跟着隐藏——和 Arc 一样「所有的地方都可以隐藏」。
- **鼠标滑到最左边缘，侧栏悬浮滑出**：不用展开也能快速用一下侧栏（点文件、切标签），鼠标移开自动缩回；`⌘\` 快捷键照旧；第一次点收起按钮会弹气泡教你快捷键。
- **超大文件夹止血（P0）**：选了个巨大的文件夹（比如整个家目录、根目录）不再把 app 直接冻死——添加高危路径（`~`、`/`、`/Users`、整个卷）前先劝导确认；扫描有条目预算，超限的文件夹标「过大」而不是无限扫下去；启动时逐个文件夹加载、不再因为上次的大文件夹卡在空白；新增「管理文件夹…」逃生门，树挂了也能把问题文件夹移掉。

## v0.10.1 — 2026-07-16

**更新体验三连修：重启真重启、更新不再要密码、进度条不再闪。**

- **点「重启安装」现在真的会重启**：此前点了它，窗口只是被藏起来、app 没退出也没安装，得自己手动退出 app 才能装上——根因是漏接了系统更新专用的退出信号，已修。
- **更新不再每次要密码/指纹**：如果 app 曾被以管理员身份安装过（这会让之后每一次更新都要授权），现在装更新时会提示做一次性修复——授权这最后一次，把 app 归属改回你，以后更新永久免密。
- **下载进度框不再闪烁**：下载更新时进度面板此前每秒拆掉重建数次导致狂闪，现在只原地更新数字和进度条，焦点也不再被反复抢走。
- **快捷键更好学了**：侧栏/顶栏主要按钮悬停显示快捷键提示（简洁版），新用户还有一次性的快捷键教学气泡。
- **侧栏拖不坏了**：侧栏最小宽度收紧到 240px，拖到最窄时顶排图标不再被裁掉。
- 说明：装上本版的那次更新走的还是旧版更新器（可能仍需手动退出一次）；**从本版起的下一次更新**开始，就是一键重启 + 免密 + 不闪的完整体验。

## v0.10.0 — 2026-07-15

**能往文档里插图片了。**

- **三种方式插图**：从访达（Finder）直接把图片拖进文档、粘贴截图（Cmd+V）、或用斜杠菜单 `/图片` 打开选择器挑图。剪贴板里同时有文字和图片时优先粘文字（从 Word/网页复制不会意外变成一张截图）。
- **图片自动优化**：插入时自动压缩——超大图缩到长边 1600 像素、手机照片方向自动摆正，避免一张原图把文档撑爆。图片以内联方式存进 `.html` 文件本身，文档依然是**自包含的单个文件**，发布或搬移不用另外带图片文件夹。
- **给图片加说明**：选中图片后在块菜单点「加说明」，可在图下方写一行说明文字；把说明清空就变回纯图片。
- **图片整块选中**：点一下图片就整块选中（光标不会插进图里），可删除、可拖拽重排；深色模式下选中框是清楚的蓝色边框。
- **新增「视图」菜单 + 两个快捷键**：`⌘\` 聚焦/切换侧栏、`⌘R` 刷新当前网页标签。
- 开发侧：图片走 Schema 校验器既有的 `<img>` / `<figure>` 合规规则，存盘字节仍是干净的标准 HTML；配套单元测试 + e2e 真门（含「存盘字节过校验器判合规」与变异自检）。远期（Phase 2）大图外置存储待需求信号再做。

## v0.9.0 — 2026-07-15

**深色模式来了,外加一大批文件管理与浏览器的稳定性修复。**

- **深色模式(浅色 / 深色 / 跟随系统)**:默认跟随系统明暗,也能在右上 ⋯ 菜单、菜单栏「外观」或设置里手动三选一。整个界面(侧栏/标签/顶栏/菜单/起始页…)全量变暗;文档内容用智能反色变暗——普通浅色文档变暗、图片和本身就是深色的文档不动,而且**深色只影响屏幕显示,存盘字节和浅色时一模一样**;支持深色的网站会自动跟着变暗;PDF 导出始终是文档原本的浅色。选择跨重启记住。
- **文件夹能拖拽移动了**:侧栏里直接把文件夹拖到别的文件夹/根目录来移动(此前只能拖文件),拖进它自己里面会被拦住;树滚动后被吸顶的文件夹标题也能当拖放目标了。
- **删除可以连续撤销**:连删多个文件时每个都有自己的「撤销」——不再是删第二个就把第一个的撤销顶掉。
- **误删正在编辑的文档能救回**:在外部(Finder 等)删掉一个正开着、有未保存改动的文档,不再静默丢改动——弹「保存到哪里」把改动救下来。
- **网页出错后能恢复**:网页加载失败出错误页后,在地址栏换一个好网址(或点刷新)现在能真恢复,不再卡在错误页上非得切标签才好。
- **一批文件管理修复**:改名不再叠出「文件.md.html」这种双后缀;文件夹的展开/收起状态重启后记住了;外置盘拔了再插回、或文件夹路径改回来会自动恢复,不用手点「重新定位」;两个文件夹里的同名文件在标签区能区分(显示所属文件夹);外部新建的文件夹默认收起、与其他文件夹一致;撤销删除后置顶状态也一起回来。
- **一批浏览器修复**:地址栏输 `note:开会` 这类带冒号的词现在走搜索、不再被当非法协议拦掉;打字打到一半切标签,残留的字不会在新标签上误打开;收藏文件夹重名自动加序号(与导入一致);导出书签再导回不再让文件夹翻倍;文档标签上那个点不动的收藏星标去掉了。
- 开发侧:新增「AI 探索测试」体系(让 AI 当用户在真 app 里找 bug)+ 对抗审查流程,本版这批文件树/浏览器修复即由它产出并逐条针对性验证(含变异自检);外观模式配套 ui-demo 对齐 + 验收门。

## v0.8.4 — 2026-07-14

**链接能跨文件夹空间了,外加一批 Wendi 反馈的修复。**

- **跨文件夹空间互链**:侧栏同时打开多个文件夹时,一个文件夹里的文档现在能链接到**另一个文件夹**里的文档——`@` 菜单直接搜到别的空间的文档(按空间名分组)、从侧栏拖文件跨空间建链、点开、悬停预览、断链红线修复全都跨空间工作;**把文件改名/移动后,别的空间里指向它的链接也自动跟着改**(可撤销)。此前文档互链只在单个文件夹内成立
- **浏览器不再反复弹人机验证**:网页标签的浏览器标识里带了 Electron/app 字样、被一些网站当异常反复要求验证——现在归一成普通浏览器标识
- **大文件夹不再卡顿**:打开或浏览大文件夹时,磁盘变化的处理从「每次扫整棵树」改成「只扫变化的部分」,消掉加载卡顿
- **添加大文件夹有加载反馈**:选了个大文件夹后读盘那几秒不再像卡死,改成两段式加载提示
- **点标签页定位到文件、但不再乱滚视口**:点一个标签,侧栏会展开到并高亮对应文件,但不再把整棵树滚走
- **隐藏/垃圾文件不进文件树**:Windows 和云盘同步产生的隐藏垃圾文件(.DS_Store / Thumbs.db / 冲突副本等)不再显示在侧栏
- **去掉编辑器里多余的蓝框**:基础编辑模式下鼠标悬停时那个蓝色虚线框去掉了(响应视觉反馈)
- 开发侧:跨根互链方案文档 + robustness 补测;跨 session 协作文档更新
