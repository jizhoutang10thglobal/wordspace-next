## U2 · 行为判官双 case 闭卷校准(3/3 过)

方法:判官指令**内联**给 3 个**隔离 agent**(不给 spec、明令禁用工具查仓 / git / memory),验 prompt 是否真有牙。防考题泄露:输入还原 pre-fix 时点。

**Case A —— 预期推导能不能自己推出标准约定**
- 输入(仅此):feature 名「跨块选中后直接打字」+「块编辑器,已拖选跨越多个块」。无 spec。
- 合格线:预期清单必须含「打字应替换选区内容」。
- 结果:**过**。判官第一条即「跨块选区被删除、字符插入、多块坍缩合并为一个——核心约定:选中后打字=替换选区,跨块也不例外」。另闭卷推出 undo 单步原子性、部分选区首末块前后缀保留、空块清除、反向 / 键盘选择等价、亮暗两态高亮清除、保存 reload 落盘一致等 20+ 条;IME 维度标「需真机手测」;对「合并后保哪个块类型」等产品决策明确标「需拍板」而不脑补答案。

**Case B —— 缺失 / 收敛会不会被误报成 bug**
- 输入:feature 名「新建文档弹窗」+ 实测「模板台只有空文档一张,无其他模板」。
- B1(无 spec):**过** → 判「疑似有意设计」、`would_report_as_bug=false`、下一步「spec 缺失 → 问产品负责人」。没直接报 bug。
- B2(给「有意分歧」章节):**过** → 查到 blank-only 是 Wendi 2026-07-23 拍板收敛 → 有意不报。

结论:`references/behavior-judge.md` 的两步工作法 prompt 通过校准,未改。

## U3 · 视觉 rubric 零误报门(修 1 条后过)

方法:采当前 app(已知好样本)5 主界面(起始页 / 侧栏树 / 块编辑器 / 新建弹窗 / 命令面板)× 亮暗双主题的截图 + computed-style 快照,对静态 rubric(R1–R6)干跑找误报。

- **抓到 1 条误报**:R1(圆角语义)把新建弹窗的 `.sb-card` 模板卡、`.sb-cm-para` 范式卡(都是 `<button>` + 7px 圆角)误判为「button 非全圆丸」。**但按「纸方墨圆」,可点的选择卡片语义上是「纸」、正确用 5–10px,不是 pill 控件**。R1 判据太粗(`tag==button` 就要 pill)。**已修**:R1 改成「判据看视觉角色不看 tag,选择卡片豁免」。
- 修后 R1–R6 在 5 主界面 × 亮暗上**零误报**;R7(无假纸叠层)、R12(编辑感)截图目视干净(弹窗是白纸 + 淡 elevation 阴影、无露边假纸)。

结论:`references/rubric.md` 经此门修正 R1 后零误报。
