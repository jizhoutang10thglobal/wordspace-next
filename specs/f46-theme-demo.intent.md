# 意图卡片 · Spec 2：暗 / 亮主题切换（F46 收窄版）

**要 AI 自动做什么（一句话）**
在 spec 1 的骨架上，于底部状态栏加一个开关：点击在暗 / 亮之间切换 **app 外壳**（状态栏、窗口边框 / 背景），而**文档纸面里的颜色保持不变**。

**为什么先做这个 / 它长在哪**
这是 demo 的**第二条 spec**，跑在 spec 1 建好的骨架上，并且**自动吃到 spec 1 写进 `CLAUDE.md` 的真实教训**（如“测试要和 Electron 解耦、E2E 无显示时 skip”）——现场能指出“第二条因为第一条学到的东西，少踩了坑”。这就是 compound。

**边界（做 / 不做）**
- ✅ 做：底部状态栏一个暗 / 亮开关；切换只作用于 app 外壳；文档纸面颜色保真不被染；本 run 若撞到新坑则追加进 `CLAUDE.md`。
- 🚫 不做：反转色预览、外部 AI 改写、跨平台一致性保证、大文档（10 万字）性能、主题持久化到重启、多套主题 / 配色编辑。（全部砍掉——这是 F46 的 demo 收窄版）

**“做完”长什么样（三件可见实物）**
1. PR 开好。
2. 权威门绿：容器内 `npm test`（Vitest）退出码 0——含“切换后文档纸面色不变”的断言。
3. feature 真能用：你在 Mac 上 `npm start`，点状态栏开关，外壳在暗 / 亮间切，文档里的颜色纹丝不动。
4. 学到东西：本 run 自动复用了 spec 1 在 `CLAUDE.md` 留下的教训（compound 的"吃到"一侧现场可指）。

**运行方式**
同 spec 1：dev container 内 `claude -p /lfg` 无人值守；确认意图后走开。
