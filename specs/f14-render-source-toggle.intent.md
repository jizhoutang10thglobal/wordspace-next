# 意图卡片 · Spec 3：渲染 / 源码切换（F14 收窄版）

**要 AI 自动做什么（一句话）**
在 spec 1/2 的基础上，于底部状态栏再加一个开关：点击让 `#doc-container` 在「排版好的渲染视图」和「这份文档的 HTML 源码文本（只读）」之间整篇切换。

**为什么先做这个 / 它长在哪**
这是 demo 的**第三条 spec**，跑在 spec 1 的文档查看器 + spec 2 的状态栏上，自动吃到 `CLAUDE.md` 里 S1/S3/S4 的真实教训。尤其它会用上 **S4 刚扩出来的「VA 验内容」能力**——这条的可见效果是「内容从渲染变成源码文本」，正好考验验收门能不能验"内容"而不只是颜色。compound 现场可指：「第三条因为前面学到的、门更强了，自己产出对的成品」。

**边界（做 / 不做）**
- ✅ 做：状态栏一个渲染 / 源码开关；整篇切换；源码视图**只读**显示 HTML 源码文本；切回重新渲染；视图状态机纯逻辑解耦（`view-mode.js`）。
- 🚫 不做：源码可编辑 / 即时落盘、语法高亮、破损 HTML 降级、不识别标签保真、多文档隔离、split 双栏、大文档性能、跨平台。（F14 的 demo 收窄版）

**"做完"长什么样（可见实物）**
1. PR 开好。
2. 权威门绿：容器内 `npm test`（Vitest）退出码 0，含视图状态机断言。
3. CI e2e 绿：`va-runner` 按 VA 真验「渲染态不漏标签、源码态真显标签 `</`、可逆」，`va-selftest` 变异自检证门有牙。
4. feature 真能用：你在 Mac 上 `npm start`，点开关在排版 ↔ HTML 源码文本之间切，肉眼一眼能看出。
5. 学到东西：本 run 自动复用 S1/S3/S4 教训（含 S4 的「VA 验内容」）。

**运行方式**
同前：dev container 内 `claude -p /lfg` 无人值守；确认意图后走开。
