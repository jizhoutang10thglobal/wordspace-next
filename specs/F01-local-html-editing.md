## 这是什么

电脑里有一种文件叫 HTML 文件——网页，还有公司很多正式文档（价值观、手册、招聘页）其实都是这种文件。麻烦在于：想改它，以前得会写代码。

这个功能让你**不用懂代码，像在画布上一样直接改**。打开一个 HTML 文件，你看到的就是它本来排好版的样子。**点任意一个元素就能选中它，拖着它放到页面任意位置，拽角缩放，双击改文字**——所见即所得，跟在 PPT / Canva 里摆东西一样自由。改完一保存，文件还是 HTML 文件，只是变成了你摆好的样子。

交互上我们**照抄开源工具 heyhtml.com**（它的源码是 `github.com/joyxiaofan-beep/html-visual-editor`，MIT）。先把它那套自由画布的 UI/UX 整套抄过来跑通，再一步步按我们自己的需要改和优化。

## 能干什么

照抄 heyhtml 的完整画布编辑能力：

- **点选元素**：鼠标悬停高亮、点击选中（八个缩放手柄出现）；按 Esc 一层层往父元素选。
- **自由拖动**：选中后拖到页面任意位置，带**智能对齐线**（跟旁边元素对齐时弹出磁吸辅助线 + 像素间距标注），方向键 1px / Shift+方向键 10px 微调。
- **缩放**：八向手柄拖拽改大小。
- **改文字**：双击进入内联编辑，富文本格式（加粗/斜体/下划线/删除线、字体、字号、文字色/高亮、标题、对齐、清除格式）——这条工具栏已经做好了。
- **插入元素**：两种模式——**浮动**（绝对定位、拖到哪是哪）和**排版流**（进文档流）。可插：容器、文本、标题、表格、图片、按钮、分隔线、链接、列表、引用。
- **右键菜单**：复制、复制/粘贴/清除样式、图层前后顺序、锁定、编组、删除。
- **多选 + 编组**：Ctrl/Cmd 点多选，编组成一个整体一起搬。
- **锁定**：锁住的元素不能误移动/缩放/删除。
- **表格编辑**：插入、增删行列、改单元格（表格/图片的更细富块见 F15）。
- **图表排版**：一组数据卡片/KPI 等排版模板。
- **格式刷**：把一个元素的样式一键刷到别的元素上（连续模式）。
- **撤销/重做**：100 级历史，几乎每个动作都能撤。
- **PDF 导出**：分页导出 PDF。
- **保存**：写回原文件，输出干净 HTML（编辑器自己的标记和 UI 全剥掉）。

## 现在做到哪了

**已有**：heyhtml 式常驻富工具栏（文字格式那条）已经照抄落地、本地真机验证过；文件打开/保存写回/历史版本/撤销重做的底子都在；无损保真的序列化（存盘只剥编辑器标记、其余原样保留）也在。

**要做的大头**：把编辑器**内核**从现在的「块流（contenteditable + 块）」换成 heyhtml 的「**自由画布（选中 / 拖动 / 缩放 / 对齐线）**」。做法是新建一批画布模块（对照 heyhtml 那 ~19 个 `HVE_*` 模块逐个抄），复用现有的富工具栏 / 保真存盘 / 文件 I/O，等画布版就位后退役只服务块流的旧模块。分两期：

- **第一期（最小可用画布）**：点选 + 自由拖动 + 对齐线 + 缩放 + 内联改字 + 插入（浮动/流）+ 撤销升级到操作级。
- **第二期**：右键菜单 + 多选编组 + 图层 + 锁定 + 表格编辑 + 格式刷 + 图表排版 + PDF 导出。

参考：heyhtml.com、repo `joyxiaofan-beep/html-visual-editor`、guide `https://heyhtml.com/guide.html`。

## 算做完的标准

打开任意一份 HTML，能像 heyhtml 那样自由可视化编辑：点选任意元素、拖到任意位置（带对齐线）、缩放、双击改字、插入新元素、撤销重做；保存写回后仍是**合法、干净的 HTML**（编辑器自己的标记 / UI / contenteditable 全部剥干净，浏览器能正常打开）。**你没主动碰过的元素不会被无故破坏。**

> ⚠ 验收口径变更（2026-06-16）：旧标准是「没改的地方一字没乱」的流式保真。改成自由画布后这条**作废**——你主动拖动 / 缩放过的元素**会**被写上定位 / 尺寸样式（`position` / `left` / `top` / `width` 等），这是画布模型的**预期行为**，不算「破坏」。新口径只要求：① 存盘产物是合法干净 HTML；② 你没主动操作过的元素不被无故改动。

## 范围

只管**本地文件、单文档**的可视化编辑。多人协作、云端同步、一键发布、AI 生成/重设计——都是后面阶段的事，这里不碰。

> 已知取舍：自由定位的输出是**固定坐标、不随屏幕宽度 reflow**（这是画布模型的固有代价）。它跟将来「发布成响应式网页 / 实时协作」有张力——那两件事到各自阶段再处理，不在 F01 范围内。
