# Schema #2「分页文档」标准化排版层 — Requirements

## Summary

给 Schema #2「分页文档」加一层**标准化排版能力**：一条顶部排版工具栏 + 一批可编辑的排版控件 + 一组具名标准预设（中文国标公文/学术论文/商务 + 英文 APA/MLA）。选一个预设，一键把整份文档的排版设成该标准的全套值（纸张 / 边距 / 中西文字体 / 字号 / 行距 / 首行缩进 / 对齐 / 段间距 / 标题各级 / 页眉页脚 / 页码）；此后任何一项都能单独改，改了就脱离预设、可另存为自定义预设。这是把 schema-2 从「Word 式分页容器」推进到「像 Word 一样能按文档标准排版」的一步。**先在 ui-demo 做到 production-ready，再 align 到真 app**（本仓 ui-demo↔真 app spec 对齐制度）。

方向来自 Wendi（2026-07-24，经 Colin 转达）：分页文档的价值不只是"分页"，而是能承载**有硬性排版标准的正式文档**——公文、标书、论文、合同。中国有强制国标（GB/T 9704-2012《党政机关公文格式》），把边距/字体/字号/行距/页码全部量化；这些必须都做成**可编辑、可修改**的。工具栏因此要比 Schema #1「流式文档」复杂，更靠近 Word，但只保留跟文档标准相关的样式设置，不做完整 Word。

## Problem Frame

Schema #2 刚从 Schema #1 拆出来（PR-A~E，2026-07-23 合 main），当前的可编辑面只有**页面级**：分页开关、纸张（A4/A3/Letter/Legal）、纵横向、边距三预设+自定义 mm、页眉/页脚文字、页码开关。这些埋在文档 ⋯ 菜单的"页面设置…"弹窗里。

缺的是**文档排版级**能力。今天文档区的排版是写死的 `BASELINE_CSS`（16px/1.75/820px，见 `src/editor/blockedit.js`）——正文字体、字号、行距、段落缩进、对齐、标题各级样式**全都不可调**。这对流式文档没问题（Notion 也不给这些旋钮），但对分页文档是硬伤：一份公文如果不能设"三号仿宋、每页22行、首行缩进2字符"，它就不是公文，只是一份恰好分页的普通文档。

产品愿景直接支持这个方向：`docs/product-vision.md` 第 62 行明确"调整字体字号等基本格式由人直接完成"，第 70 行有"公司的手册、标书、演示文稿等版式保存为模板"，第 97 行导出 PDF/Word。所以标准化排版层不是新增识别面，是补齐 North Star 里早已写下的能力。

代价是真实的：让排版可调意味着（a）排版设置必须**跟着文件走**（HTML-native，不存本地），而 Schema 硬底线不许块级带 `style`、不许 head 放自定义 `<style>`——得扩现有的 `data-ws-schema-css` 白名单机制（现在只装 `@page`）再装一段排版 CSS；（b）预设点名的具体字体（仿宋_GB2312 / Times New Roman）在浏览器/真 app 不一定装得到，"国标合规"的底其实在字体上。这两点在 Assumptions/Risks 展开。

## Actors

- **A1 · 中文正式文档作者**（主力）：写公文、标书、论文、合同的人，需要产出符合国标或行业规范的文档。选"国标公文"预设 → 直接得到合规版式，微调个别项。
- **A2 · 英文文档作者**：写学术论文、报告的人，需要 APA/MLA 版式。
- **A3 · AI / Agent 作者**：调用 Wordspace 生成分页文档；预设值同时是 AI 生成合规文档的目标参数（schema-2 AI 创作指南要能表达这些）。
- **A4 · 模板复用者**：从"另存的自定义预设"或未来的模板库取用公司既有版式，快速起一份带标准样式的文档。

## Key Decisions

- **KD1 · 预设 + 微调，不是纯控件也不是纯预设。** 内置具名标准预设（国标公文 / 学术论文 / 商务 / APA / MLA），选中一键设好全套值；每个旋钮仍可单独改；改任意一项即"脱离预设"（工具栏预设名标记为"自定义"），可另存为新预设。预设是**起点不是锁死**。（Colin 2026-07-24 拍板，四选项之首。）理由：最贴合 Wendi"所有国标设置都要可编辑"+ 愿景里的模板库；纯控件让新手面对一堆旋钮无从下手，纯预设又违反"可修改"。

- **KD2 · 中英并重，v1 一次到位。** 第一版就同时做中文国标预设和英文 APA/MLA 预设，**单位 mm/inch 可切**，**中西文字体分设**（正文可分别指定中文字体和西文字体）。（Colin 2026-07-24 拍板。）理由：标准对比的价值一次展现完；数据模型从一开始就语言中立，避免"先做中文再回头改成双语"的返工。

- **KD3 · 进阶维度：标题各级 + 段间距进 v1；奇偶页/避头尾不进。** H1–H4 各级可分别设字体/字号/粗细（Wendi 专门点了 heading）；段前/段后间距可调。**不做**：奇偶页镜像页眉页脚 / 首页封面差异化（→ 国标页码"奇右偶左"v1 先用单一规则）、中文避头尾/标点禁则（先靠浏览器 `line-break` CSS 兜底）。（Colin 2026-07-24 多选拍板。）理由：标题各级是"像 Word"的关键且国标靠字体区分层级；避头尾是重渲染活、收益隐晦，浏览器已能兜大部分。

- **KD4 · 顶部排版工具栏，常用在栏、其余进弹窗。** 分页文档打开时，文档区上方出现一条排版工具栏：预设下拉 + 常用控件（中/西字体、字号、行距、对齐）。重的（边距、页眉页脚、标题各级、段间距、单位切换）进一个 ⚙"页面设置"弹窗。（Colin 2026-07-24 拍板。）理由：最贴 Wendi"工具栏"、"像 word 但精简"的原话；顶部一条物理上装不下全部控件，所以"常用在栏/其余进⚙"是硬约束不是可选（见 Assumptions）。

- **KD5 · 排版设置 HTML-native，跟文件走。** 所有排版值持久化进文档文件本身（扩 `data-ws-schema-css` 机制），不存 localStorage、不存应用配置。（延续 paged-doc 的 `@page` 入盘决策 + 愿景"文档归使用者所有"。）ui-demo 侧允许先用 localStorage 当原型脚手架（与现有 paged 原型同口径），但真 app align 时必须入盘。

- **KD6 · 只做跟标准相关的样式旋钮，不做完整 Word。** 明确不做：分栏、装订线、手动分页符、文字颜色/高亮的富文本工具、图文环绕、样式刷。工具栏的每一项都要能回答"它对应哪条文档标准设置"。（Wendi 原话"我们不需要像 word 那么多工具"。）

## Requirements

### 预设库（R1–R5）

- **R1.** 内置 ≥5 个具名标准预设：**国标公文**（GB/T 9704-2012）、**中文学术论文**、**中文商务**、**APA**（第7版）、**MLA**（第9版）。每个预设是一组完整的排版值（见附录 A 的精确初始值），选中即整份文档套用。
- **R2.** 工具栏顶部有预设下拉，显示当前预设名。选预设 → 全套排版旋钮即时更新、文档即时重排。
- **R3.** 用户改动任意一个排版旋钮后，预设名标记为"自定义 · 基于〈原预设〉"，表示已脱离预设。不弹确认、不阻止——脱离是正常操作。
- **R4.** 用户可把当前排版值"另存为预设"，取名后进入自定义预设列表，之后可在下拉里复用。（这是通向愿景"模板库"的桥；v1 只需 per-user 本地保存即可，不做跨组织共享。）
- **R5.** 预设覆盖的维度 = R6–R17 的全部。选预设一键设好，不留"预设没管到、还得手动补"的旋钮。

### 页面级（R6–R9，多数已存在，需接进工具栏/弹窗）

- **R6.** 纸张：A4 / A3 / Letter / Legal；方向：纵向 / 横向。
- **R7.** 页边距：上/下/左/右四值可调，**单位可切 mm ↔ inch**（切换时数值换算，不是重输）。保留现有三预设（普通/窄/宽）作为快捷。
- **R8.** 页眉 / 页脚：每页显示的单行纯文本（已存在，PR-C）。工具栏/弹窗里可编辑。
- **R9.** 页码：开关 + 位置（v1 简单规则，如底部居中）。国标"奇右偶左"镜像列入 deferred。

### 正文排版（R10–R15，全部新增）

- **R10.** 正文字体**中西分设**：一个中文字体选择器 + 一个西文（拉丁/数字）字体选择器；渲染时中文字符走中文字体、拉丁字符走西文字体（CSS `font-family` 回退链即可表达）。
- **R11.** 字号：**号 + pt 双显**（下拉列"三号 (16pt)"这种；中文用户习惯"号"，见附录 B 对照表）。允许直接输入任意 pt。
- **R12.** 行距：支持"倍数"（1.0 / 1.5 / 2.0）与"固定值 N 磅"两种模式（国标是固定值，英文学术是双倍）。
- **R13.** 首行缩进：以"字符数"（中文习惯，2字符）或长度（inch/mm）表达；可设 0（块间空行式段落）。
- **R14.** 段落对齐：左 / 居中 / 右 / 两端对齐。
- **R15.** 段间距：段前 / 段后间距可调（磅或行）。

### 标题各级（R16–R17，新增）

- **R16.** H1–H4 每一级可分别设：字体（中西分设同 R10）、字号、粗细。国标层级靠字体区分（一级黑体/二级楷体/三四级仿宋），APA 靠对齐+粗细+斜体区分——数据模型要能表达"每级一组独立样式"。
- **R17.** 标题样式随预设一键设好；单级可脱离预设单独改（同 R3）。

### 工具栏形态（R18–R20）

- **R18.** 顶部排版工具栏**仅在分页文档（schema-2）打开时出现**；流式文档（schema-1）不显示（它没有这些旋钮）。
- **R19.** 顶部栏放常用：预设下拉、中/西文字体、字号、行距、对齐、⚙入口。其余（边距、页眉页脚、标题各级、段间距、单位切换、另存预设）进 ⚙"页面设置"弹窗。
- **R20.** 任一控件改动**即时生效、即时重排**，每页仍严格 = 一张纸（延续 paged-doc V4 分页铁则）。

### 持久化与合规（R21–R23）

- **R21.** 全部排版值持久化进文档文件（KD5），通过 Schema 校验器（扩 `data-ws-schema-css` 白名单，不放宽块级 `style` 禁令）。round-trip：设置→存盘→reparse→重新打开，排版原样恢复且文档仍 conform。
- **R22.** 排版设置只对合规文档可用（非合规/降级文档不显示工具栏，同现有 paged 入口门控）。
- **R23.** 分页产物（推挤 padding / spacer）绝不进持久化（strip-on-persist，延续 paged 铁则）；排版设置进持久化。两者分清。

## Key Flows

- **F1 · 套用国标公文。** A1 新建/打开一份分页文档 → 顶部工具栏预设下拉选"国标公文" → 整份文档即时变成 A4/上37下35左28右26mm/三号仿宋/固定28磅行距/首行缩进2字符/两端对齐/标题黑体楷体仿宋分级/页码 → 作者直接开始写正文，版式已合规。
- **F2 · 微调后另存。** A1 在国标公文基础上把正文字号从三号改成小三 → 预设名变"自定义·基于国标公文" → 点"另存为预设"取名"本公司公文" → 下拉里出现该自定义预设，下次一键复用。
- **F3 · 中英切换感知。** A2 选"APA"预设 → 边距单位显示为 inch、正文 Times New Roman 12pt 双倍行距左对齐 → 若切"国标公文"，单位变 mm、字体变仿宋、对齐变两端。同一套控件承载两种标准。
- **F4 · 单位切换。** A1 在弹窗把边距单位从 mm 切到 inch → 四个边距值按换算显示（37mm→1.46in），不清零、不要求重输。
- **F5 · AI 生成合规文档。** A3 调 Wordspace 生成一份"国标公文" → 生成的 HTML 自带正确的排版 `data-ws-schema-css` 块 → 打开即分页、即合规、工具栏预设显示"国标公文"。

## Acceptance Examples

- **AE1.** 打开任一分页文档，顶部出现排版工具栏；打开流式文档，不出现。
- **AE2.** 选"国标公文"预设，导出 PDF：纸张 A4，四边距 37/35/28/26mm（±1），正文三号仿宋，每页正文行数≈22，首行缩进2字符，两端对齐——可量。
- **AE3.** 改正文字号后，工具栏预设名从"国标公文"变为"自定义"字样；文档立即按新字号重排且每页仍=一张纸。
- **AE4.** 单位从 mm 切 inch，边距 37mm 显示为 1.46in（四舍五入到设定精度），文档版式不变（只是显示单位变了）。
- **AE5.** 设置一份文档存盘 → 关闭 → 重新打开：排版原样恢复，且 `node scripts/validate-schema.js` 判其 conform、classify 为 schema-2。
- **AE6.** 选"APA"预设，正文为 Times New Roman 12pt 双倍行距、首行缩进 0.5in、左对齐（右边缘不齐）。
- **AE7.** H1–H4 在"国标公文"预设下分别呈现黑体/楷体/仿宋的字体差异（可查 computed font-family）。

## Scope Boundaries

**范围内（v1）**：R1–R23 全部。中英双预设、中西文字体分设、mm/inch 双单位、标题各级、段间距、顶部工具栏、HTML-native 持久化。

**Deferred（留口，非本期）**：
- 奇偶页镜像页眉页脚 / 首页（封面）差异化 → 国标页码"奇右偶左"降级为 v1 单一规则。
- 中文避头尾 / 标点禁则 → 先靠浏览器 `line-break: strict` / `word-break` 兜底。
- 跨组织共享模板池（愿景模板库的公开池）→ v1 只做 per-user 本地自定义预设。
- 真 app 字体内嵌（真合规所需）→ v1 ui-demo 用系统字体替身。

**Outside this feature's identity（明确不做）**：
- 完整 Word 工具集：分栏、装订线、手动分页符、文字颜色/高亮富文本、图文环绕、样式刷、修订/批注。
- 每个控件都要能对应一条"文档标准设置"，否则不进（KD6）。

## Dependencies / Assumptions / Risks

- **RISK-1 · 字体可用性 = "国标合规"的真正的底。** 预设点名"仿宋_GB2312""Times New Roman"是具体字体文件。浏览器（ui-demo）和 Electron（真 app）不保证装了——仿宋_GB2312 是 Windows 字体，mac 上没有。**ui-demo 决策**：用最接近的系统 CJK 字体当替身（宋体/SimSun、PingFang、系统仿宋 STFangsong 等），视觉≈但非逐字节国标；预设里记录"意图字体名 + 回退链"。**真 app 后话**：要真合规可能得内嵌字体（体积/授权问题），列入 align 阶段评估。这条不解决不影响 v1 交付，但要对用户诚实表达"版式合规，字体尽力"。

- **RISK-2 · 排版进文件的 schema-legal 机制需要 plan 阶段设计。** 方向已定（KD5：扩 `data-ws-schema-css`，不放宽块级 style 禁令），但具体 CSS 形状、白名单校验规则、strip 契约是 ce-plan 的活。关键约束：① 排版 CSS 只能编辑器托管、不许用户手写任意 CSS（否则打穿 Schema 安全边界）；② 校验器对磁盘字节 reparse 判（§4.3 铁律）；③ 与现有 `@page` 块并存。

- **ASSUMPTION-1 · 顶部一条装不下全部，切分是硬约束。** 中西两个字体下拉 + 单位 + 预设 + 字号 + 行距 + 对齐已经很挤。"常用在顶部、其余进⚙"（KD4/R19）不是设计偏好，是空间事实。ui-demo 排布时以此为前提。

- **ASSUMPTION-2 · 各校论文/商务标准差异大，取通用默认。** "中文学术论文""中文商务"没有单一国标（学位论文的 GB/T 7714 主要管参考文献格式，版式靠各校规定）。附录 A 给的是合理通用默认，标注"可调"，不宣称"唯一正确"。国标公文和 APA/MLA 有权威来源，那三个是硬值。

- **DEP-1 · 依赖 paged-doc V4 分页引擎。** 本 feature 是排版层，坐在已有的分页/推挤引擎上（`ui-demo/src/lib/page.ts` + `Canvas.tsx`；真 app `src/lib/schema-page.js` + `pagination.js`）。排版值改变行高/边距→引擎重算页数，两者的交互（改字号后每页仍=一张纸）是核心验收点（AE3/R20）。

- **DEP-2 · 与 schema-2 AI 创作指南对齐。** 预设值同时是 AI 生成合规分页文档的目标（A3/F5）。`docs/schema-2-ai-authoring.md`（PR-E 已落）需在实现后补上排版块的写法，否则 AI 生成的分页文档排版是 baseline 默认、非标准。

## Outstanding Questions（交给 ce-plan / 实现时定，非阻塞）

- OQ1. 排版 CSS 在文件里的确切形状（一个 `data-ws-schema-css="type"` 块？分正文/标题多块？），及校验白名单规则。
- OQ2. "字号号↔pt"下拉的具体档位（只列常用 号，还是全 16 档 + 自由输入）。
- OQ3. 中西文分设字体在 CSS 层的实现（`font-family` 回退链 vs `:lang()` vs unicode-range），哪种在本 contenteditable + 分页下最稳。
- OQ4. 自定义预设的本地存储形态（ui-demo localStorage；真 app 存哪——用户目录配置？还是也进某个共享文件？）。

## 附录 A · 标准预设初始值（ui-demo 建预设直接用；权威值来自调研，通用默认可调）

> 单位：mm 为主，inch 预设用 inch。字号中文用"号"（pt 见附录 B）。

### A.1 国标公文（GB/T 9704-2012，硬值，权威）

| 维度 | 值 |
|---|---|
| 纸张 / 方向 | A4（210×297mm）/ 纵向；版心 156×225mm |
| 边距 上/下/左/右 | 37 / 35 / 28 / 26 mm（±1） |
| 正文字体 中/西 | 仿宋_GB2312 / Times New Roman |
| 正文字号 | 三号（16pt） |

## 附录 C · 中英排版关键差异（做双语控件时要处理的）

1. **纸张**：A4（公制，√2 比例可对折）vs Letter 8.5×11in（英制）。国标锁 A4，英文默认 Letter。
2. **单位**：mm vs inch。同一"边距"控件双单位（R7/F4）。
3. **字体两套**：中文需 CJK 字体（宋体/仿宋/黑体/楷体）且中西文可分设（R10）；英文只有拉丁字体。
4. **行距语言**：中文说"固定 N 磅 / 每页 22 行"，英文说"双倍行距 2.0"（R12 两种模式都要）。
5. **段落**：中文首行缩进 2 全角字符；英文首行缩进 0.5in 或块间空行（R13）。
6. **对齐**：中文公文两端对齐撑满；英文学术左对齐留毛边（MLA 明确要求不两端对齐）（R14）。
7. **标点/避头尾**：中文全角标点 + 行首禁则；英文半角。（v1 deferred，靠 CSS 兜底。）
