# 分页可编辑性重构（ui-demo）— 2026-07-09

Colin 实测 5 个问题，根因归两条 + 一个产品决策。

## 诊断（ce-debug）

**根 1：分页用「运行时改 contenteditable 的 DOM」实现 → 从根上和编辑打架。**
`applyInnerSplits`（Canvas.tsx）为了让超高块「块内切分」，往活的可编辑 DOM 里：
- 表格插 `<tr class="ws-inner-split">` 占位行；
- `<pre>` 里 `splitText()` 再插 `<span>`；
- 元素加 runtime `margin-top`。

后果：
- **#1 代码 / #5 表格无法编辑**：内容被注入垃圾节点、光标/选区被打乱。
- **#4 点一下列表两页合并**：effect 有「编辑中的块不切」(`b.id === editingId`)——一点进超高块它就撤掉切分、两页合并。
- 每次 input 都 clear→测量→重挂，churn 巨大。

**根 2：#3 页间空白是 `pointer-events:none` 的死区**，不是可点内容，所以点不进去编辑。

**产品决策：#2 删掉分页符。**

## 对 #2 的判断（直说）

Word **有**手动分页符（Ctrl+Enter）。但它不会让「一页长一页短」——插入分页符后当前页仍是**整张 A4**，只是提前结束、底部留白，页的**物理尺寸不变**。所以你说的「必须每页统一长宽」这个前提，分页符其实并不违反。

**但我同意删**，理由不同：分页符给分页逻辑引入一整类边界（空尾页、页界处的分页符），复杂度高、又不是「像 Word 自动分页」这个核心价值的必需件。删掉能让健壮性故事干净很多。以后要加回来很简单。→ **删。**

## 方案（ce-plan）：分页 V3 — 零内容 DOM 改动

铁律：**分页层永远不碰 contenteditable 的内容 DOM。** 只做「测量 + 装饰覆盖层 + 流内间隔」。

1. **内容 = 普通可编辑流**，每个块始终是可编辑元素，任何时候都能点进去改（表格/代码/列表同）。分页层绝不注入/切分/改内容节点。
2. **测量**算出分页 Y（contentH 的整数倍），**吸附到最近的「安全切线」**：顶层块之间的间隙，或超高块内部 tr/li/代码行的边界——保证没有元素被横切。
3. **块边界处的分页**：插一个**流内间隔**（现 PageGap，改成点击可路由到相邻块）把下一块推下去 → 得到真正的上下页边距（Word 式干净分页）。
4. **超高块内部的分页**（块本身 > 一页，如 44 行表/深嵌套列表/长代码）：**只画覆盖层的页界带**（吸附到行/项/代码行边界），内容连续流过、**不注入不切分** → 表格/列表/代码跨页但完整可编辑（像 Word 里跨页的表格）。
5. **覆盖层**：纸边/页码 chip/页界带，绝对定位、`pointer-events:none`、按测量算出——纯装饰。
6. **空白/间隔点击路由到最近块**（修 #3 死区）。
7. **编辑即重排**：input → rAF 重算。因为不改内容 DOM、间隔/覆盖层不影响块内光标，所以不丢光标、不合并。
8. **删分页符**：块类型/斜杠项/breakAfter/相关测试/「分页符炸弹」种子文档一并去掉。

## 验收
每份 edge-case 文档开分页后：
- 表格/代码/列表**任意块可点可编辑**，改长度后分页**即时且正确重排**；
- 点超高块**不合并**；点页间空白能进最近块编辑；
- 每页物理高 = 一张纸；行/项/图不被页界带横切；
- 巨图轰炸：单张超页高图仍整图推页（不劈）。
