# Arc 的交互到底是怎么做的——以及 Wordspace 怎么抄

> 2026-07-08 · Wendi 说浏览器这块「抄 Arc」。这份调研把 Arc 桌面版的交互拆成 5 个域(侧栏标签模型/命令优先/瞬时浏览/内容创作融合/视觉动效),逐个 pattern 判「已有/该抄/改造着抄/不抄」。判定用三把尺子:①我们的定位(浏览为文档服务)②Arc 自己的死因(新奇税——官方数据 Spaces 只有 5-12% 用户在用)③纸方墨圆设计语言(抄魂不抄皮)。给 Wendi 汇报用,完整判定表在附录。

## 一句话:Arc 的灵魂

**内容永远是主角,chrome 能藏就藏;所有东西按寿命分层——临时的默认会消失,想留的必须给一个明确动作。**

Command Bar(一个键唤出全部入口)、Auto Archive(12 小时不碰自动归档)、Little Arc/Peek(外链弹小窗看完即走)全是这句话的投影:**进得快、退得快、留下有仪式**。

它死的原因也在同一句话里:它把这套哲学包装成一堆要学的新名词(Space、Easel、Boost、Air Traffic Control),学的多、赚的少,被自己的"新奇税"压死。所以抄的纪律很简单:**凡是骑在普世肌肉记忆上的润滑,原样抄、连键位一起抄;凡是 Arc 自造的概念容器,一律不抄,魂另找载体。**

## 我们比 Arc 多一样东西:文档

Arc 的「留下」最多是钉一个标签、归档一个 URL;**我们的「留下」是存成一份落盘的、可编辑的、进文件树的文档**——归宿强得多。所以 Arc 的瞬时浏览三件套在我们这应该收敛成**一个手势**:

> 文档里点外链 → Peek 浮层看一眼 → 要么 **Esc 走人**(零残留),要么 **「存为文档」**(接上已做好的 Readability 剪藏),要么**升级成网页标签**。

弹层、Esc、保存,全是旧动作,用户不用学任何新名词。Auto Archive 的魂只施于网页标签(不钉就不持久),**文档标签一根手指都不碰**——文件树本身就是我们的 Archive 和 Library。

## 上一轮已经抄完的(Wendi 可核对)

侧栏三层结构(置顶/标签页/文件 ≈ Favorites/Pinned/Today,位置即状态)、无地址栏(面包屑)、Cmd+P 文件模糊搜、Cmd+T 新建、Cmd+\ 真收起(chrome-less 纯内容模式)、临时文档(= Arc Notes 的正确做法,我们的笔记生来是一等文档;Arc 收官时自己都在淘汰内置 Notes)、防重复开标(keyOf 单一真相)、微交互密度(纸方墨圆动效一等公民)。

## 这一轮值钱的:4 个小润滑 + 2 个大件

**小活(普世键位,零学习税,合计一两周)**
1. **Cmd+Shift+T 重开最近关闭的标签**——误关即救,关闭栈上限 20,半天活。
2. **Cmd+Shift+C 拷贝当前文档路径**(网页=清洗后的 URL)——Arc 被夸最多的标志性小交互,我们用户把路径贴给终端/Slack/AI 是真高频,现在完全没有键盘路径。半天活。
3. **Ctrl+Tab MRU 文档切换器**——快按回上一个、按住循环;写作+参考横跳是编辑器第一场景,现在只能摸鼠标。纯文字白卡浮层,Arc 被骂的 5 个上限/花哨缩略图全不抄。中活。
4. **Cmd+P 升级成文件+命令混排**——输入「>」列动作(新建/另存为/导出 PDF/收起侧栏…),把散在菜单里的既有动作收进一个入口;文件永远第一梯队,不做 Arc 式几十条命令的深井。中活。

**大件(跟 browser-tabs 合流排期)**
5. **Peek 外链浮层 + 「存为文档」出口**——上面说的那个手势,融合主线的临门一脚,战略价值最高。
6. **Split View 两栏分屏**——Arc 全部功能里唯一零差评的大件(拖拽+分屏是旧心智,不收学习税);「左资料右文稿」直接服务定位。第一期只做两栏+hairline 分隔线(纸方墨圆:一条细线,不做花哨手柄),建议单独立 spec。

## 明确不抄的(和为什么)

**Spaces**(Arc 自己的数据 5-12% 使用率=新奇税头号铁证;我们刚在 #118 拆掉 Space 概念,不开倒车;彩渐变与纸方墨圆直接冲突)、**Folder 标签分组**(文件树已承担组织职责;Arc 需要它是因为没有文件系统,我们有)、**Easels 画板/活截图**(小圈子真爱、大众摆设,「截片段留着用」的魂已由剪藏承接且落盘更可靠)、**Air Traffic Control 路由规则**(要用户配规则表,适用前提都不存在)、**强制 Auto Archive 文档标签**(收走用户摆的工作状态是惊吓,Arc 官方都承认难适应)、**彩色渐变/颗粒感的皮**(评测原话 twee/fisher-pricey,两极;我们守暖灰纸面)。

## 附录 B:Arc 交互细节速查(实现时用)

- **Auto Archive**:默认空闲 12h 归档(可调不可关);归档≠删除,Library>Archived Tabs 可搜可恢复。官方自己承认"一开始很难适应"。
- **Command Bar**:Cmd+T 悬浮居中,混排 已开标签/历史/动作;"已开标签优先于新开"防重复。
- **Little Arc**:外部 App 链接弹无侧栏小窗,Cmd+O 收编进主窗;小窗自己也 6h 自动归档。口碑:理念好、"记住要用这个快捷键本身是负担"。
- **Peek**:从 Favorites/Pinned 点外链先弹预览浮层,Esc 即走、可升级成完整标签。
- **Split View**:拖标签到边缘吸附/Cmd+点,最多四格,分隔线连续调宽;唯一全员叫好的大件。
- **视觉**:每 Space 主题渐变、网页圆角卡片浮在彩底上、无工具栏(Cmd+Shift+D 按需召唤看完整 URL)。皮的口碑两极:设计圈爱,大众评"twee/fisher-pricey"。
- **键位速查**:Cmd+T 命令栏 / Cmd+D pin toggle / Cmd+S 收侧栏 / Cmd+Shift+T 重开 / Cmd+Shift+K 清空临时 / Cmd+Shift+C 拷链接 / Ctrl+Tab MRU / Cmd+Option+N Little Arc。
