00 · WHAT CHANGED

v2 改了什么,观点是什么

Colin 的反馈:组件略普通。v2 的回答是一套贯穿所有组件的设计语言——「纸方墨圆,按了会陷,叠了有边」。以下五条,每一条都能在 02 节亲手摸到。

  1. 纸方,墨圆。承载内容的「纸」(卡片/菜单/弹窗)保持微圆角; 所有会动的「墨」(按钮/开关/分段控件)一律变成圆丸。可点的和可读的,一眼分开——这是 v1 "全员圆角矩形"最缺的辨识度。
  2. 层级不再靠边框,靠纸的物理。卡片与菜单底下露出真实的叠纸边 (多层硬投影),弹窗背后垫两张微倾斜的纸;边框大幅退场,留白与层差接管分隔。
  3. 按了会陷(letterpress)。按钮 hover 抬 1px、按下沉 1px, 脚下的硬投影同步伸缩;kbd 键帽同理能按。位移只有 1–2px,但手指能感觉到。
  4. 焦点是圈选,不是发光。全站 :focus-visible = 2px 墨青蓝外圈 + 3px offset,像在纸上圈重点;拒绝模糊 glow。输入框例外——它的焦点表达是一根 320ms 从左划过的墨线。
  5. hover 语汇统一为「墨条滑入」。菜单、右键菜单、侧栏、命令面板,同一根 3px 墨条从左缘生长 + 底色淡入。另新增四个件:segmented / switch / 右键菜单 / ⌘K 命令面板, 全部纯 CSS 可交互,动效收进 120/200/320ms 三档规范。

01 · MOTION SPEC

动效与手感规范

三档时长、四条曲线,全部收进 CSS 变量。原则:越贴手指越快(按压 120ms),越大的纸越慢(弹窗 320ms);spring 只给"物"(开关滑块、置顶卡),不给颜色。

把文档放回自己的电脑里

汪迪 · 2026 年 6 月 30 日 · 产品手记本地优先

过去十年,我们把笔记搬进了一个又一个云端应用,也把它们锁进了一个又一个私有格式。服务关停、订阅涨价、导出残缺,同样的故事每隔几年就重演一次。Wordspace 想做的事情很简单:你的文档,就是你磁盘上的一个 .html 文件,任何浏览器都能打开,五十年后依然如此。

这不是复古,而是把「所有权」当成产品的第一性原理——编辑器可以换、公司可以消失,但文件必须一直属于你。

为什么是 HTML

HTML 是唯一同时满足三个条件的格式:开放标准、人类可读,并且自带排版能力。Markdown 足够简单,但表格、待办和嵌套结构都要靠方言;PDF 是打印品不是文档;私有 JSON 则把内容锁在应用里。我们选择 HTML Living Standard 的一个受限子集,细节见 docs/html-kernel.md。

  • 任何浏览器双击即读,不需要安装任何软件
  • Git 能 diff,全文检索工具能直接索引
  • 五十年后的解析器,仍然认识它

我们给自己的三条底线

  1. 文件永远以标准 HTML 存盘,不写任何私有标记
  2. 断网时所有功能可用,同步只是锦上添花
  3. 卸载 Wordspace 之后,你不损失任何内容
