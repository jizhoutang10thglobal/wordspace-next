# 块编辑器 · 跨块选中与替换 —— 对齐 spec

> 占位 spec（2026-07-22 补建）。块编辑器整体尚无 spec，本份只覆盖**跨块选中 / 跨块删除 / 跨块打字替换**这一块——即 Wendi 2026-07-22 一系列视频反馈(bug3/bug4/bug5/bug6 + 跨块打字)落地的那片交互。斜杠菜单、转为、拖拽手柄、格式气泡等其余块编辑行为待各自被碰到时另建。

## 行为契约

**单块编辑 vs 跨块选中是两种态。** 点进一个块 = 单块编辑态，光标在块内，打字/删除/退格交原生浏览器处理。拖选跨过块边界、或按两次 ⌘A 选全篇 = 跨块选中态：选区罩住 ≥2 个顶层块。

**跨块选中的视觉（对齐 Notion）。** 选区罩住的每个顶层块**整行给蓝底**，不是只高亮零碎文字片段——用户要能一眼看清「选中了哪几行」。单块内的选区维持原生文字高亮不变，只有跨 ≥2 块才切整行蓝底。选区折叠 / 点别处 → 蓝底立即清除。（来源 #314）

**跨块删除 / 退格 / 剪切。** 对跨块选区按 Delete/Backspace 或剪切：把选区罩住的块内容删掉、并把首尾残块并成一个干净块，**不留空 `<ul></ul>` / 空块死尸**（删完点不进、删不掉的幽灵块是 bug——bug4/bug6）。删空整个待办列表后，后续段落的位置/勾选框不能错位遮挡（bug6 追加）。

**列表块行首退格并入上一块。** 列表首项行首按 Backspace，光标要能上移并入上一块，而不是「什么都不发生」（bug3）。

**跨块选中后直接打字 = 替换。** 跨块选中态下敲一个字符键：删掉整个跨块选区、把该字符插进一个干净块（替换语义，跟单块内选字打字覆盖一致）。**英文字母已支持**（#324）。

**跨 toggle 边界删除一致化（U23/select-4）。** 选区端点**部分跨 toggle（`<details>`）边界**（一端在 toggle 体内、另一端在外，或分属不同 toggle）→ 删除**一致化为空操作**：绝不「删 toggle 外那侧、体内不动」的半应用（半应用与选区高亮承诺不符、且零反馈）。判据 = `detOf(startContainer) !== detOf(endContainer)`（`detOf` = 最近 `details` 祖先）。**toggle 被选区整体包含**（两端都在其外、`detOf` 相等且为 null）→ 不拦、维持现状可整删（⌘A 全篇删、跨段整删 toggle 不回归）。空操作时给**可感知反馈**：当前跨块高亮的块闪一下橙红（CSS 动画 `ws2-nope`，随 `data-ws2-rangesel` 高亮视觉语言走，不推布局/不用 transform/不用系统 beep——app 无音频反馈先例）。**本单元只做一致化 + 反馈**，结构正确的完整跨 toggle 删除语义属 deferred（见有意分歧）。

**⚠ 已知限制见「欠账」——跨块选中态打中文目前唤不起输入法。**

## 有意分歧

- **编辑器架构本质不同**（历史形成，两侧独立演进）：ui-demo `BasicEditor` 是**单个 contenteditable** 区域，原生选区天然跨"块"、原生删除天然可用，所以 ui-demo 侧没有、也不需要这套跨块机制。真 app 块编辑器是**每个块一个独立 contenteditable**（保真 + 块级操作的地基），跨块选区被原生的块边界钉死，才需要 `wallDropped` 摘墙 + `deleteSelection` 自删 + `data-ws2-rangesel` 自绘高亮这一整套。**因此本 spec 的行为契约主要约束真 app 侧**；跨块相关 bug（bug3/4/5/6 + 打字）也都是真 app 专属。
- 谁拍板/日期：架构分歧属既有事实，未经单独拍板；本占位 spec 首次把它记录在案（2026-07-22）。若日后要让两侧编辑器架构收敛，另开 brainstorm。

## 欠账

- **跨块选中态打中文唤不起输入法**（Colin 2026-07-22 拍板暂不修）。病根：macOS 只在焦点元素**可编辑**时才激活中文输入法；跨块选中态下编辑块的 contenteditable 已被摘掉、没有任何可编辑元素被 focus → 输入法不出来 → 打拼音必出英文。#324 的 keydown 拦截只能替换已经变成英文的字母，治不了 IME。治本需让跨块态有个可编辑输入宿主，但焦点进可编辑元素会塌掉原生选区，而删/退格/剪切/打字整套现读原生选区判断作用域——要一并改成读 `rangeSelEls`（独立于会塌的原生选区），是对跨块机制的精细重构，且真 IME 只能真机验、自动化测不了。**用户绕法**：想用中文重打替换，先点进某个块（单块编辑态、输入法正常）再删改。
- **完整的跨 toggle 删除语义**（结构正确地删掉部分跨界选区、既不半删也不整个吞 toggle）属 deferred（U23 只做「一致化为空操作 + 反馈」）：要在保 toggle「summary + ≥1 体块」合规不变式的前提下，正确裁剪跨 details 边界的选区，是独立的结构删除设计，留作后续。用户绕法：想删 toggle 内外相连的一段，分两次删（先删 toggle 外、再点进体内删）。
- **块编辑器其余行为（斜杠菜单 / 转为 / 拖拽手柄 / 格式气泡 / 粘贴闸）尚无 spec**，本份只覆盖跨块选中一片，其余待各自被碰到时补。
