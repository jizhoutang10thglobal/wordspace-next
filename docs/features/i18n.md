# 系统级中英切换（i18n）—— 对齐 spec

> 状态：ui-demo Phase 1 实现中（`feat/ui-demo-i18n`）。真 app 移植 = Phase 2（见欠账）。
> 需求：Colin 2026-07-15（中/英双版 + 切换；为未来多语言把架构提前想好；先 ui-demo 后真 app）。

## 行为契约

### 三态语言偏好（与外观三态同构）

用户偏好 `LangPref = 'system' | 'zh' | 'en'`，生效语言 `Lang = 'zh' | 'en'`：

- **跟随系统（默认）**：按操作系统语言——ui-demo 读 `navigator.language`，真 app 读 `app.getLocale()`；
  以 `zh` 开头 → 中文，其余 → English（英文兜底，未来加语言在 `langOfSystem` 扩）。
- **中文 / English**：锁定，不随系统。

⚠ **系统语言无监听事件**（不像深浅色有 `matchMedia('prefers-color-scheme')`）：只在启动 / 切换偏好时读一次系统 locale，
跟随系统模式下改了系统语言要**重开页面**才生效。这是刻意限制（浏览器/Electron 都不给系统 locale 变更事件），别造监听。

### 切换入口（拍板②）

**只在设置页**一个语言选择器（照外观选择器先例，不加顶栏常驻钮——语言是一次性设置，不值得占顶栏）。
选项：跟随系统（随 UI 语言显示）/ 中文 / English（后两者恒母语显示，选错语言也找得回自己的语言）。
切换即时生效（zustand store 变更 → React 全树重渲染，无刷新）；`<html lang>` 同步设 `zh-CN`/`en`。

### 翻译范围（拍板④，Phase 1 只翻 UI chrome）

| 翻 | 不翻 |
|---|---|
| 按钮 / 菜单 / toast / 对话框 / 空态 / tooltip / 设置页 / 斜杠菜单 / 快捷键面板 / 起始页 | **用户文档内容**（永不） |
| 磁盘默认名（新建文档/文件夹，见下） | **模板正文**（模板台的会讯/周报模板正文——Phase 2，涉 Wendi 产品面） |
| | **mock 演示数据**（假网站/种子文档/演示书签，只存在于 ui-demo） |

### 磁盘默认名（拍板③）

新建文档/文件夹的默认名**按当前语言生成**并写进磁盘：英文态新建 → `Untitled.html` / `New Folder`；
复制副本后缀 `副本` → ` copy`。铁律：① **已存在的文件永不改名**（那是用户数据，只有*新建*时用当前语言默认名）；
② 查重后缀（`未命名文档 2` / `Untitled 2`）对英文名同样成立，别写死中文名再拼数字。

### fallback 规则

`t(key)`：en 缺 key → 回 zh（半翻译界面能用，绝不显示裸 key 名）；zh 缺 key → 显示 key 名（开发期「漏建字典」的可见信号）。

## 架构（为什么不用 react-i18next）

终点是真 app：**vanilla JS renderer**（`src/renderer/*.js`，无 React）+ **Electron 主进程**（菜单栏 `Menu` /
原生对话框标题 / 主进程推的 toast 全要翻）。React 生态 i18n 库进不了主进程和 vanilla renderer。故自研微型层：

- **纯逻辑**（`src/i18n/core.ts`）：`normalizeLangPref` / `effectiveLang` / `makeT`，不 import React/DOM/zustand →
  可原样搬进真 app 两个进程（本仓「纯逻辑可移植」传统：theme-manager / tabs.js / appearance 先例）。
- **字典按命名空间分文件**（`zh/{common,sidebar,editor,browser,modals,settings,shortcuts,misc}.ts` + `en/` 同构）：
  ①多人/多 PR 改不同模块不撞同一文件；②将来加日语 = 加一个 `ja/` 目录，零架构改动。key 在文件里不带命名空间前缀，
  `index.ts` 合并时加（`t('sidebar.openFolder')`）。zh = 源语言（原文照搬）。
- **React 外壳**（`src/i18n/index.ts`，ui-demo 独有）：zustand store（localStorage `ws-language`）+ `useT()` 响应式 hook
  + `t()` imperative（给 store/lib 等非 React 调用方）+ `initLang()`（首屏设 `<html lang>`）。

## 防漂移门：CJK 扫描（i18n 的"脊梁"）

`ui-demo/scripts/i18n-scan.mjs`（`npm run i18n:scan`）：用 **TS AST** 只报字符串字面量 / 模板串 / JSX 文本 / JSX 属性里的
中日韩字符。**代码注释天然不进 AST → 仓库的中文注释惯例不受影响**（这是必须用 AST 而非裸 grep 的原因）。
豁免两条：① 文件级白名单（`i18n/`、`mock/seed.ts`、`mock/pagedSamples.ts`、`lib/nonConformSamples.ts`、
`components/MockSites.tsx` 等演示/种子数据）；② 行内 `// i18n-exempt`（个别确不该翻的常量）。
**报红 = 门失败**。这条门让「以后任何人新写硬编码中文当场被拦」，是「不用后面再加很多代码」的真正保障。

|---|---|---|
| 纯逻辑核心 | `ui-demo/src/i18n/core.ts` | `src/lib/i18n.js`（搬 core，CI 门覆盖的 canonical） |

## 有意分歧

ui-demo 用 mock 数据、React；真 app 用真文件系统、vanilla renderer + Electron 主进程。i18n 纯逻辑两侧同款（改逻辑两处一起改），
外壳因技术栈不同各写一份——同 appearance / theme-manager 的先例，不算漂移。

## 欠账

- **Phase 2 = 真 app 移植**：core 搬 `src/lib/i18n.js`；renderer 双份（vanilla）+ **Electron 主进程**（菜单栏 `Menu`、
  原生对话框、主进程 toast 如「Wordspace 浏览器不支持下载」、`app.getLocale()`、settings 持久化）；照外观 U4-U7 的移植路径。
- **模板正文双语化**（Phase 2，Wendi 参与）：模板台的会讯/周报等模板正文出英文版。
- **真 app e2e 全按中文文案断言**：port 时**测试环境锁 zh**（启动 env 固定语言），新断言优先 `data-testid` 而非文案。
- **mock 演示数据不翻**：假网站/种子文档保持原样（白名单已豁免）。
- **英文文案偏长**（如「置顶」→"Pin to top"）：窄列 truncate 是否兜住，发现挤爆记已知项，不在本 feature 修布局。
- **翻译质量**：英文第一版由 AI 出，**Wendi 终审术语口径**（PR 附术语表）。
