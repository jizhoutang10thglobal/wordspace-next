# 行为预期清单 · 新建文档弹窗(判官第一步产出,采证前定稿)

纯从 feature + spec + 现代编辑器常识推导,未看实现/运行行为。

**A. 三入口打开**
- A1 标签区「+」→ omni 弹窗(地址栏+范式轨+pane),temp 临时态不落盘。
- A2 Cmd+T = 标签区「+」(同一 omni 弹窗)。
- A3 文件夹 hover「+」→ 非 omni(标题头「新建文档·在{文件夹}」+X,无地址栏)。
- A4 文件夹右键「新建文档」= hover「+」(非 omni,定点落盘)。
- A5 非 omni 标题头「{位置}」= 用户点的真实文件夹名,跟随变化。
- A6 三入口范式轨都在。

**B. 范式轨**
- B1 恒三档:类 Notion(当前)/分页文档/范式 3(灰锁),各带图标+名+描述+轨底注。
- B2 打开默认选中类 Notion,右 pane 直接显模板卡(不需先点)。
- B3 点分页文档 → pane 切分页模板卡,选中态移过去。
- B4 点范式 3(灰锁)→ 不建东西,pane 变占位(锁圈+「范式3·还在路上」),无卡。
- B5 范式 3 不可获焦/不可 Enter 误建(键盘也跳过)。
- B6 范式切换是纯切换,不关弹窗/不建/不丢态。
- B7 选中态视觉清晰。

**C. 模板卡**
- C1 类 Notion=1「空文档」卡;分页=1「空白分页文档」卡。
- C2 pane 只显该范式对应 schema 模板,不串范式。
- C3 点空文档卡 → 按模板建,默认名「未命名」,弹窗关,新文档打开可编辑。
- C4 omni/temp 建的临时态不落盘;非 omni 直接落盘到标题头文件夹。
- C5 点空白分页卡 → 分页视图打开,磁盘 schema-2(meta=2 + @page)。
- C6 卡片键盘可达(Tab 到、Enter/Space 建)。
- C7 卡片 hover 反馈。
- C8 快速连点同卡只建一个(点完即关),不重复/不留幽灵弹窗。

**D. 关闭(弹层通行)**
- D1 Esc → 关、不建、无副作用。
- D2 点遮罩 → 关、不建。
- D3 点 X → 关、不建(omni 的 X 在地址栏右、非 omni 在标题头)。
- D4 关后焦点回合理位置,不丢到 body。
- D5 所有关闭方式都不误建/误改文件树(P1 级:取消必须干净)。

**E. omni 地址栏(仅 temp)**
- E1 打开焦点自动落地址栏,可直接打字。
- E2 输网址/搜索 Enter → 开网页标签导航 + 关弹窗,不建文档。
- E3 区分网址 vs 搜索词(**具体阈值 spec 未写死 → 待核,不当 finding**)。
- E4 空地址栏 Enter → 什么都不发生。
- E5 聚焦墨线动效亮暗两态都可见不刺眼。
- E6 omni 下范式轨+pane 并存(两条路都在)。

**F. 焦点/键盘**
- F1 打开建立 focus trap(Tab 在弹窗内循环,不跑背后)。
- F2 纯键盘走完全程(开→Tab 移→Enter 建→Esc 取消)。
- F3 焦点顺序符合视觉(左轨→右 pane→关闭)。
- F4 非 omni 无地址栏,首焦点落合理处(第一卡/第一可用范式)。

**G. 亮暗主题**
- G1 全弹窗两态对比度达标、无不可读、无暗态刺眼白卡。
- G2 灰态范式 3 两态都明显更暗/淡,一眼分出不可用。
- G3 范式选中态高亮两态都清晰。

**H. 相邻 feature 交界**
- H1 temp 建的临时文档 = 未保存标签,不污染文件树(存盘前树里没有)。
- H2 非 omni 落盘建后,文件树立即出现新文档(正确文件夹下)。
- H3 建文档与全局 ⌘Z 的关系合理(**落盘新建能否被 ⌘Z 撤销属产品决策 → 待核**)。
- H4 弹窗打开背后被模态遮住不可交互。
- H5 连续新建每次干净独立,不叠弹窗/不残留旧选中态。

**I. 中文 IME(需真机手测,自动化不验)**
- I1 omni 地址栏 IME composition 期间 Enter 只上屏候选、不提交导航。
- I2 新建文档打开后能正常中文输入,首字符不丢。

**待核(拿不准/需 Colin 定,不当 finding)**:E3 网址vs搜索阈值;H3 落盘新建能否 ⌘Z 撤销;范式 3 键盘是「彻底跳过」还是「可聚焦不可激活」。
**有意分歧(spec 已声明,不报)**:真 app 无 /templates 模板管理页、模板 blank-only(每范式一张空卡)。
