## 4. 六个单独项（Colin 已拍板，穿插做；多为 bug/行为改动，不在视觉审计里）

> ⚠ 下面几条上个 session summary 有过期/存疑描述，已在 `2ac4dcd` 上复核。带「需实证」的，改前先真机复现、别照抄我的假设。

1. **新文件没页边距** — 现象：新建/临时文档没有页边距，已存过的真文件有。**强假设（需实证确认）**：临时文档走 `loadFromHtml`→`srcdoc`（shell.js:352），srcdoc **继承父窗口的严格 CSP**（style-src 无 unsafe-inline），文档里的 baseline `<style id="ws-schema-baseline">`（`max-width:820px;margin:0 auto;padding:48px 60px`，见 demo-clean.html / `ensureSchemaBaseline`）被 CSP 拦掉 → 无边距；`file://` 载入不继承 CSP → 有边距。**注意**：`blockedit.js:156-167` 已用 adoptedStyleSheets 注 `EDITOR_CSS`（CSP-safe），但那套**不含 body 页边距**（页边距是"入盘"内容、走持久 `<style>`）。**修法方向**：把 body 页边距也加进 adoptedStyleSheets（CSP-safe、live 生效），持久 `<style>` 保留给序列化。**先确认**：真开 app 新建文档，DevTools 看 srcdoc 里那条 baseline `<style>` 是否被 CSP block（console 应有 CSP 报错）。
2. **PDF 缩放 >100% 不变** — 根因：`shell.css:392 .pdfv-page{max-width:100%}` 把画布显示宽度封顶；`pdf-viewer.js:84` `renderAll` 设 `canvas.style.width=w1*s`，但被 CSS max-width 卡住。修法：缩放态（`fitMode=false`）时不封 max-width，让画布溢出 + stage 横向滚动。
3. **临时文档默认名 → 「未命名」+ 存盘可编辑文件名** — 现在临时文档用 `template.base` 当名（`sidebar.js:913` `openCreateModal`/`openSaveModal` 用 `t.base`）。改：临时文档默认名「未命名」；SaveModal（`sidebar.js:509`）里加可编辑文件名输入框，存盘时用它。
4. **保存到外部文件夹** — SaveModal（`openSaveModal`，`sidebar.js:509`）现在只列工作区文件夹。加「浏览…/其他位置」钮 → 弹 macOS 原生保存框（app 已有原生 dialog 能力，走 ipc / preload；找 `src/main/ipc.js` 现有 export/pickFile 的 dialog 用法照抄）。保留快速存工作区的自定义 SaveModal。
5. **自动保存 + 已保存 ✓** — **注意**：`✓ 已保存` 提示**已经有了**（`shell.js:50 flashSaved`，保存成功闪绿字~1.6s）。要做的是：真文件（有 docPath）改动后 debounce 自动存盘；临时文档仍需显式选位置存。审计 #75 另有一条：临时文档存成功后补 toast 正反馈（`doSaveTemp` 成功后 `showToast`）。
6. **Cmd+Z 撤销** — **需实证**：撤销对**块编辑器是接了的**（`shell.js:104` `undoMgr=new WS2Undo.UndoManager`、`:127` Cmd+Z handler、`undo.js` 是 WS2Undo）。用户报"Cmd+Z 不起作用"要先复现是**哪种文档**：(a) 基础编辑器（非合规文档，`attachBasic`/`basic-edit.js`）—它的 keydown（shell.js:147 附近）处理了 bold 等、**撤销可能没接**（要看是否漏了 undo，基础编辑器用 execCommand、原生 undo 可能失效）；(b) 块编辑器 WS2Undo 自身有 bug。**先定位是哪种编辑器、哪种操作 Cmd+Z 失效**，再修，别默认某一侧。

## 5. 环境 / 验证 / 工作流（血泪坑，务必读）

- **真机验证是唯一权威**：vitest 只测脱离 Electron 的纯逻辑，碰不到 renderer/preload/CSP/主题 apply。视觉/交互对齐必须**真开 app** 或走 e2e。e2e 真跑只能在 CI（GitHub Actions + xvfb）或宿主 macOS，dev container 里跑不了。

- **ui-demo 参照**：`http://localhost:5180/`（React dev），源在 `ui-demo/`。对齐时对照它，但**别 port 它的 React 组件结构**，只搬视觉参数（色值/尺寸/圆角/阴影/间距）。
- **push 身份**：推本仓要用 `jizhoutang10thglobal` 账号（`CTlandu` 403）：`gh auth switch --user jizhoutang10thglobal` 推完切回。**main 有分支保护**（require branches up-to-date + required e2e/test check），PR 合前必先并最新 main + 等 CI 重跑，`--admin` 绕不过 required check。
- **worktree**：并发 session 各开 worktree，别在共享目录切分支劫持工作树。本分支现在在主库 `wordspace-next-demo` 检出；若有并行 session 要用主库，另开 worktree。
