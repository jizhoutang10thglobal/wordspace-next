# [P3-03] 内联改名强制保留原后缀:输「火箭.md」落盘成「火箭.md.html」

**问题(2/2)**:对 `xx.html` 内联改名输入 `火箭.md` → 落盘 `火箭.md.html`。后端 `renamePath`
(workspace.js)始终拼回原 `ext`,把用户输入整体当 base。安全(永不丢后缀)但反直觉,还造出双后缀怪名。

**修法(小步,不开放任意改后缀)**:改名输入**恰好以已知文档后缀结尾**(`.html/.htm/.md`)且与原后缀不同时,
视为「显式换后缀」意图——但 v1 **不做真转换**(md↔html 是格式转换不是改名,乱改会坏文件),而是:
剥掉用户输入里的后缀再拼原后缀(`火箭.md` → `火箭.html`),同时 toast 提示
「改名不改格式;要转 Markdown 用『另存为/导出』」。输入以**原后缀**结尾则照旧剥重复(`火箭.html`→`火箭.html`
不出 `.html.html`——先验现状,若已处理就只做前半)。非文档后缀(`.txt`)继续当 base 的一部分,维持现状。

**门**:单测(workspace.js rename 已有测试文件,追加 4 例:同后缀重复/异文档后缀/非文档后缀/无后缀);
变异:去掉剥后缀逻辑→翻红。

**spec 记账**:workspace-file-tree.md 行为契约「改名不改格式」一句。
