# [P3-02] 置顶标签的关闭钮:spec 与实现打架 —— **已拍板:方案 B(Colin 2026-07-14「该有关闭钮」)**

**现状(已核实)**:所有标签(含置顶区)都渲染 ×(sidebar.js ≈:1808);置顶区的 × 走 `removeTabRel`
(= 取消置顶并移除),普通区走 `closeTabRel`。这是 v0.4.5 有意加的(「置顶区×关闭」,当时的拍板)。
而浏览器 spec §4.4 写「置顶标签无关闭钮」(Chrome 惯例:钉住=防误关)。两个先例打架,差异没进 §13 差异表=漂移。

**拍板项(Colin 选一)**:
- **方案 A(spec 赢)**:置顶区去掉 ×,想关先取消置顶。好处=防误关(钉住的本意);代价=多一步操作,
  且改掉 v0.4.5 以来的既有习惯。
- **方案 B(实现赢,推荐)**:保留 ×,把「置顶有 ×(点击=取消置顶并移除)」写进 spec §4.4 + §13 差异表。
  理由:已上线半个多月的习惯、当初是有意为之、⌘W 对置顶已有保护(不误关),鼠标 × 是显式动作误触率低。

**执行(按已拍的 B)**:纯 docs,代码零改动——`docs/browser-feature-spec.md` §4.4 改成
「置顶标签有关闭钮,点击=取消置顶并移除(⌘W 仍不关置顶)」+ §13 差异表/§15 决策日志补一行
(拍板人 Colin,2026-07-14);`docs/features/browser.md` 欠账区若引了这条,同步清掉。

**门**:无代码门(docs-only)。
