# [P3-10] 导出 → 再导入,非书签栏文件夹翻倍 —— **拍板复议项,先问再动**

**现状(2/2,行为与拍板一致、不是实现 bug)**:导入重名文件夹「加后缀不合并」是 §4.9 拍板#6(2026-07-10)。
后果:用户「导出当备份 → 恢复导入」这个常见心智下,自己的「工作」文件夹变成「工作」+「工作 2」翻倍
(书签栏本身有特判去重,其他文件夹没有)。探索员用 Chrome 真实导出文件和自导出往返都撞到。

**拍板项(Colin 选一)**:
- **维持拍板**:什么都不改,本计划关闭(风险=用户恢复备份后收藏翻倍,体感差)。
- **温和修正(推荐)**:保持「不合并」原则,但**内容完全相同**的文件夹(名同 + 内含书签 url 集合相同)
  视为「重复导入同一份」→ 跳过不再造副本;内容有差异才加后缀。这样「导出→原样导回」零翻倍,
  「合并两份不同收藏」仍不丢东西。判定纯逻辑,落 bookmarks.js。

**执行(若选修正)**:bookmarks.js 导入函数加「同名文件夹内容指纹比对」;单测:原样往返零新增/
有差异加后缀/空文件夹同名跳过。变异:去掉指纹比对→翻红。spec §4.9 拍板#6 更新措辞+日期。
