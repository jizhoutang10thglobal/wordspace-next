# [P3-09] 收藏里手动建/改文件夹允许同名(导入路径反而会去重)

**问题(2/2)**:收藏管理页连点「新文件夹」→ 出仨「新文件夹」;把文件夹改名成已存在的名字也直接生效。
数据层按 id 走不会坏,但收藏区分组标题、「移动到」下拉里视觉分不清;而**导入**路径对重名是「加后缀 2」
——同一个 app 里两套口径。

**修法**:把导入的去重函数(bookmarks.js 里现成,grep 后缀「 2」的 uniqueName 逻辑)复用到手动路径:
`bm-add-folder` 和文件夹 rename 的 IPC 处理里,名字撞车时自动出「<名> 2」。不弹错误(低摩擦,和导入一致)。
纯逻辑落 `src/lib/bookmarks.js`(单测可测),IPC 只调用。

**门**:单测(test/bookmarks.test.js 追加):连建两个「新文件夹」→ 第二个叫「新文件夹 2」;改名撞车同款。
变异:去掉去重调用→翻红。

**spec 记账**:browser-feature-spec.md §收藏管理(文件夹命名口径与导入一致)。
