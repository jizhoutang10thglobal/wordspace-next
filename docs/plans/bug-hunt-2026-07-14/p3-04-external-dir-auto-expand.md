# [P3-04] 外部新建的文件夹在实时树里默认展开,与其它文件夹不一致

**问题(2/2)**:app 开着,外部 `mkdir` 一个新文件夹 → 树里它**展开**着出现;而 app 内建的/重启后的
文件夹都默认收起。不一致还带副作用:探索员曾因此差点误报「子文件不显示」(点了一下反而收起)。

**根因方向(执行时核实)**:collapsed 集合按 rel 记「被用户收起过的」,外部新增的 rel 不在集合里 →
渲染成展开。app 内建文件夹走的路径大概率显式加了 collapsed(对照 `wsMakeDir` 的收尾)。

**修法**:reconcile/patch 后发现「新出现的目录 rel」(新树有、旧树无)→ 默认加进 collapsed(=收起),
与 app 内建、重启行为对齐。注意别把「改名的目录」(ino 同、rel 变)误收起——改名要继承原展开态,
外部改名目录的展开态继承现状先实测,别顺手改坏。

**门**:e2e(live-tree.spec.js 追加):外部 mkdir+塞文件 → 断言该目录行是收起态(无子行渲染);
外部改名已展开的目录 → 仍展开。变异:去掉默认收起→翻红。

**spec 记账**:workspace-file-tree.md 一句「新目录默认收起,外部/内建一致」。
