# [P3-06] 两个根里的同名文件在标签/置顶区完全分不清

**问题(2/2)**:根 A、根 B 各有 `a.html`,都打开 → 标签区两条都只显示「a.html」,置顶同理,
连 tooltip 都没有(title 空)。数据层(data-root)是分的,纯显示问题。命令面板已经做对了(显示「根名 / 文件名」)。

**修法(照 VS Code 收敛)**:
1. 低成本先做:所有工作区内标签行加 `title` tooltip = `根名 / rel`(一行代码,普适有益)。
2. 消歧后缀:**仅当**打开的标签里存在「同名不同根」冲突时,冲突各方的标签文字加淡色小字后缀
   「 — 根名」(样式照纸方墨圆的 `--c-text-3`)。无冲突不加(别把所有标签搞长)。冲突检测在
   renderZones 里按 basename 分组一遍即可,量小。
3. 置顶区同款。外部标签(↗)已显示 abs tooltip,不动。

**门**:e2e(multi-root spec 追加):双根同名都打开→断言两标签文字带各自根名后缀;关掉一个→后缀消失。
变异:去掉冲突检测→翻红。

**spec 记账**:workspace-file-tree.md(或 tabs 相关 spec,grep 哪个管标签)加「同名跨根消歧」契约。
